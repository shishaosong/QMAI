# 第三方自定义模型 403 错误修复说明

## 问题描述

使用第三方自定义模型（如 https://hub.linux.do）时，"拉取模型"和"测试模型"总是返回 HTTP 403 错误。

## 问题根源

在 `src/lib/llm-providers.ts` 的 `withCustomOriginHeader` 函数中：

```typescript
// 修复前的代码
export function withCustomOriginHeader(headers: Record<string, string>, url: string): Record<string, string> {
  return {
    ...headers,
    ...(isLocalOrPrivateHttpEndpoint(url) ? localLlmOriginHeader() : { Origin: "" }),
  }
}
```

**问题：** 对于非本地端点（公网中转站），代码设置了 `Origin: ""`（空字符串），导致某些中转站的安全策略拒绝请求，返回 403 Forbidden。

## 修复方案

修改 `withCustomOriginHeader` 函数，对非本地端点**完全不设置 Origin header**，而不是设置空字符串：

```typescript
// 修复后的代码
export function withCustomOriginHeader(headers: Record<string, string>, url: string): Record<string, string> {
  // 本地端点需要 Origin: http://localhost 来通过 Ollama 等的 CORS 检查。
  // 非本地端点（公网中转站）：完全不设置 Origin header，让浏览器/Tauri 使用默认行为。
  // 之前设置 Origin: "" 会导致某些中转站返回 403 Forbidden。
  if (isLocalOrPrivateHttpEndpoint(url)) {
    return {
      ...headers,
      ...localLlmOriginHeader(),
    }
  }
  // 非本地端点：不添加 Origin，让请求使用默认值
  return headers
}
```

1. **src/lib/llm-providers.ts** - 修复主要逻辑
2. **src/lib/llm-providers.spec.ts** - 更新单元测试（不再期望 `Origin: ""`）
3. **src/lib/settings-model-list.spec.ts** - 更新单元测试（不再期望 `Origin: ""`）

## 技术细节

### 为什么不能用 `Origin: ""`？

1. **空字符串不是有效的 Origin 值** - HTTP 规范中 Origin 应该是完整的 URL 源（如 `http://localhost`）或 `null`
2. **中转站的安全策略** - 某些中转站会拒绝包含无效 Origin 的请求
3. **Tauri 的行为** - 虽然注释说"Tauri HTTP 插件在 unsafe-headers 下会把空 Origin 视为显式移除"，但实际上空字符串可能被发送出去

### 为什么这样修复有效？

1. **本地端点** - 继续使用 `Origin: http://localhost`，保持与 Ollama 等本地服务的兼容性
2. **远程端点** - 不设置 Origin header，让 Tauri/浏览器使用其默认行为，通常会：
   - 自动设置合适的 Origin
   - 或者完全不发送 Origin（对于简单请求）
   - 中转站接受这种标准行为

## 影响范围

此修改影响所有使用"自定义模型"的场景，包括：

- ✅ 第三方中转站（OpenAI 兼容接口）
- ✅ Anthropic 兼容接口
- ✅ Responses API 接口
- ✅ 本地 Ollama 服务（不受影响，继续使用 localhost Origin）
- ✅ 本地 LM Studio（不受影响）

## 回滚方案

如果修复导致问题，可以回滚到之前的版本，恢复 `Origin: ""` 设置。但根据测试，这不太可能需要。
