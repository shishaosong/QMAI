### **需求1: 停止分析功能** ✅

#### 实现内容
1. **Store层** - 添加 `AbortController` 支持
   - 在 `BookAnalysisTask` 接口添加 `abortController` 字段
   - 修改 `startTask` 方法接收 `AbortController` 参数
   - 新增 `cancelTask` 方法触发停止操作

2. **启动逻辑** - 创建并传递 `AbortController`
   - 在 `handleStartAnalysis` 中创建 `AbortController`
   - 将 `abortController.signal` 传递给分析引擎
   - 正确处理取消异常

3. **UI层** - 停止按钮
   - 在运行中的任务卡片添加"停止分析"按钮
   - 点击后调用 `cancelTask` 触发停止
   - 任务状态自动更新为"用户取消分析"

#### 技术实现
```typescript
// Store中的取消逻辑
cancelTask: (taskId: string) => {
  set((state) => {
    const task = state.tasks.find((t) => t.id === taskId)
    if (task?.abortController) {
      task.abortController.abort()  // 触发 AbortSignal
    }
    return {
      tasks: state.tasks.map((t) =>
        t.id === taskId
          ? { ...t, status: "error", error: "用户取消分析", updatedAt: Date.now() }
          : t
      ),
      currentTaskId: state.currentTaskId === taskId ? null : state.currentTaskId,
    }
  })
}
```

## 📝 使用说明

### 停止分析功能
1. 启动一个拆书分析任务
2. 在分析进行中，任务卡片会显示进度条和"停止分析"按钮
3. 点击"停止分析"按钮
4. 分析立即停止，任务状态变为"用户取消分析"

### 主导航集成
1. 在左侧导航栏找到拆书分析图标（书本图标）
2. 点击进入拆书分析视图
3. 左侧边栏会显示当前导入的作品名称
4. 如果导入"大奉打更人"，侧边栏会显示"大奉打更人"而不是"章节"
