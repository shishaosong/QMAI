# 打包遇到的问题与建议

## 🐛 问题描述

**错误**：`LINK : fatal error LNK1104: 无法打开文件 xxx.dll`

**原因**：
- 有进程仍在占用 Rust 编译产物（DLL 文件）
- 可能是之前的编译进程没有完全退出
- Windows 文件锁导致链接器无法访问文件

---

## 💡 建议的解决方案

### 方案 1：重启应用（推荐）⭐

1. 关闭所有相关进程
2. 重启 VS Code 或开发环境
3. 然后再打包

### 方案 2：手动释放文件锁

1. 打开任务管理器
2. 结束所有 `cargo`、`rustc`、`node` 相关进程
3. 等待 10-20 秒
4. 再次打包

有时文件锁会自动释放，等待 5-10 分钟后再试。

## 🔍 技术细节

### 文件锁问题的根源
Windows 系统中，当一个进程打开文件时会创建文件锁。即使进程结束，文件锁可能还会短暂保留。这在 Rust 编译时特别常见，因为：
1. 编译器会生成临时 DLL
2. 链接器需要访问这些 DLL
3. 如果上次编译没有完全清理，文件就被锁定

### 为什么 cargo clean 也失败
因为某些 DLL 文件仍被占用，`cargo clean` 无法删除它们（error 5: 拒绝访问）
