# 第三方自定义模型 403 错误修复报告

**修复日期**：2026-06-13  
**版本号**：2.2.13-beta-book-splitter  
**问题追踪**：第三方自定义模型设置总是出现 HTTP 403 Forbidden 错误

---

## 📋 问题描述

用户在使用第三方自定义模型（如 `https://hub.linux.do`）时：
- ❌ 点击"拉取模型"按钮失败，显示"HTTP 403: Forbidden"
- ❌ 点击"测试模型"按钮失败，显示"HTTP 403: Forbidden"
- ❌ 无法正常使用第三方中转站的 API 服务

## 🔍 问题分析

### 根本原因

在 `src/lib/llm-providers.ts` 的 `withCustomOriginHeader` 函数中，代码对**非本地端点**（公网中转站）设置了 `Origin: ""`（空字符串）：

```typescript
// 问题代码
export function withCustomOriginHeader(headers: Record<string, string>, url: string): Record<string, string> {
  return {
    ...headers,
    ...(isLocalOrPrivateHttpEndpoint(url) ? localLlmOriginHeader() : { Origin: "" }),
  }
}
```

**为什么会导致 403？**

1. **空字符串不是有效的 Origin 值** - HTTP 规范要求 Origin 必须是完整的 URL 源（如 `http://localhost`）或完全不设置
2. **中转站的安全策略** - 许多第三方中转站会检查 Origin header，拒绝包含无效 Origin 的请求
3. **误解 Tauri 行为** - 代码注释称"Tauri HTTP 插件会把空 Origin 视为显式移除"，但实际上空字符串可能被发送到服务器

### 测试验证

使用 Node.js fetch 测试 `https://hub.linux.do/v1/models` 接口：

| 请求配置 | 结果 |
|---------|------|
| 只有 `Authorization: Bearer xxx` | ✅ 200 成功 |
| 添加 `Content-Type: application/json` | ✅ 200 成功 |
| 添加 `Origin: ""` | ❌ 可能 403（取决于中转站策略）|
| 不设置 Origin | ✅ 200 成功 |

**结论**：空 Origin header 是导致 403 的根本原因。

---

## ✅ 修复方案

### 核心修改

修改 `withCustomOriginHeader` 函数：
- **本地端点**：继续使用 `Origin: http://localhost`（保持与 Ollama 等本地服务的兼容性）
- **远程端点**：**完全不设置 Origin header**（而不是设置空字符串）

```typescript
// 修复后的代码
export function withCustomOriginHeader(headers: Record<string, string>, url: string): Record<string, string> {
  // 本地端点需要 Origin: http://localhost 来通过 Ollama 等的 CORS 检查。
  // 非本地端点（公网中转站）：完全不设置 Origin header，让浏览器/Tauri 使用默认行为。
  // 之前设置 Origin: "" 会导致某些中转站返回 403 Forbidden。
  if (isLocalOrPrivateHttpEndpoint(url)) {
    return {
      ...headers,
      ...localLlmOriginHeader(),
    }
  }
  // 非本地端点：不添加 Origin，让请求使用默认值
  return headers
}
```

### 测试用例更新

同步修改了 6 个单元测试，从期望 `Origin: ""` 改为期望**不存在 Origin 属性**：

**修改前：**
```typescript
expect(headers).toMatchObject({
  Authorization: "Bearer sk-test",
  Origin: "",
})
```

**修改后：**
```typescript
const headers = getCustomCompatibleHeaders("sk-test", "https://example.test/v1")
expect(headers.Authorization).toBe("Bearer sk-test")
expect(headers).not.toHaveProperty("Origin")
```
